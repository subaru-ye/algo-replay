# 算法复盘 — 产品需求文档

## 1. 产品概述

### 1.1 产品定位

面向算法练习者的个人复习管理工具。核心解决一个问题：**刷过的算法题，如何科学地安排复习，避免遗忘。**

产品以「间隔重复」为核心理念，根据每道题的掌握程度自动生成复习计划，并通过每日看板推送当日任务，帮助用户在最佳时间点进行复盘巩固。

### 1.2 目标用户

- 准备算法面试的求职者
- 保持算法手感的在职开发者
- 系统性刷题的学生或竞赛选手

### 1.3 产品形态

纯前端单页应用（SPA），数据存储在浏览器本地，无需后端服务。后续可扩展云端同步能力。

## 2. 功能需求

### 2.1 P0 — 已完成

| 模块 | 功能 | 说明 |
|------|------|------|
| 题目管理 | 录入题目 | 填写题号、题名、刷题日期、掌握程度 |
| 题目管理 | 编辑题目 | 修改已有题目的信息 |
| 题目管理 | 删除题目 | 确认后删除，不可恢复 |
| 题目管理 | 题目列表 | 按日期分组展示，支持展开/折叠 |
| 复习调度 | 间隔重复 | 五种掌握程度对应不同复习间隔（见下表） |
| 每日看板 | 今日任务 | 展示当日待复习题目列表 |
| 每日看板 | 完成复盘 | 更新某道题的掌握程度，重新计算下次复习日期 |
| 数据统计 | 刷题趋势 | 近 30 天刷题数量柱状图 |
| 刷题记录 | 尝试记录 | 每道题记录最多 5 次尝试的颜色块 |

#### 掌握程度与复习间隔

| 等级 | 标识色 | 含义 | 复习间隔 |
|------|--------|------|----------|
| 毫无头绪 | 红色 | 完全不会，看题解才能写出来 | 1 / 3 / 7 / 14 天后 |
| 有印象没写出 | 橙色 | 记得思路但写不出完整代码 | 2 / 5 / 10 天后 |
| 磕绊做对 | 琥珀色 | 反复调试才勉强做对 | 3 / 7 / 14 天后 |
| 独立做对 | 蓝色 | 比较顺利地完成，偶尔卡壳 | 7 / 14 天后 |
| 信手拈来 | 翡翠绿 | 快速准确独立完成，完全掌握 | 无需复习 |

## 3. 非功能需求

### 3.1 性能

- 首屏加载 < 2s（Lighthouse Performance > 90）
- 1000 道题目以内列表渲染无明显卡顿

### 3.2 兼容性

- 支持 Chrome / Firefox / Edge / Safari 近 2 个主版本
- 支持移动端响应式布局

### 3.3 数据安全

- 所有数据仅在用户本地浏览器存储
- 删除操作为软确认后执行
- 导出/导入功能提供数据备份能力

### 3.4 可维护性

- 组件使用 shadcn/ui 规范，保持一致性
- 业务逻辑集中在 `hooks/` 和 `lib/` 中，UI 组件保持纯净
